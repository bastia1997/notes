50 cumulative exercises: from syntax to Spring Batch

50 mixed problems to consolidate what you've learned. Basic syntax, OOP, collections, streams, concurrency, JDBC/JPA, Spring Core, REST, Security, Spring Batch. Hidden solutions.

Java Foundations (1-10)

Ex 1 — Atomic counter Thread-safe counter with `AtomicInteger`, test with 10 threads × 100k increments. 
class Counter {
    private final AtomicInteger n = new AtomicInteger();
    public void inc() { n.incrementAndGet(); }
    public int get() { return n.get(); }
}
Ex 2 — Unicode palindrome `isPalindrome(String)` ignoring case, spaces and accents. Hint: `Normalizer.normalize(s, NFD).replaceAll("\\p{M}", "")`.
Ex 3 — Record sorting `record Person(String name, int age)`. Sort a list by age desc, then name asc. 
people.sort(Comparator.comparingInt(Person::age).reversed()
    .thenComparing(Person::name));
Ex 4 — Generic Pair swap 
public record Pair<A, B>(A a, B b) {
    public Pair<B, A> swap() { return new Pair<>(b, a); }
}
Ex 5 — Custom exception `InvalidIbanException extends IllegalArgumentException` with `code` and `reason`.
Ex 6 — Record equals/hashCode Create `Money(BigDecimal amount, String currency)` record and verify `equals`/`hashCode` are auto-implemented.
Ex 7 — Defensive copy Make `Period(LocalDate from, LocalDate to)` truly immutable. (LocalDate is immutable, so `final` fields suffice.)
Ex 8 — Bitwise: count set bits 
int n = 0b10110011;
System.out.println(Integer.bitCount(n));   // 5
Ex 9 — Switch expression `monthName(int m)` with arrow switch.
Ex 10 — Sealed types for validation 
sealed interface ValidationResult permits Ok, Error {}
record Ok() implements ValidationResult {}
record Error(String message) implements ValidationResult {}

String msg = switch (v) {
    case Ok ok -> "valid";
    case Error e -> "invalid: " + e.message();
};

Collections, Stream, IO, Concurrency (11-20)

Ex 11 — Top 3 cities by population 
cities.stream()
    .sorted(Comparator.comparingInt(City::population).reversed())
    .limit(3)
    .forEach(System.out::println);
Ex 12 — Group by with sum 
Map<LocalDate, BigDecimal> daily = orders.stream()
    .collect(Collectors.groupingBy(Order::date,
        Collectors.reducing(BigDecimal.ZERO, Order::total, BigDecimal::add)));
Ex 13 — Word count from file 
try (Stream<String> lines = Files.lines(Path.of("text.txt"), UTF_8)) {
    Map<String, Long> wc = lines
        .flatMap(l -> Arrays.stream(l.toLowerCase().split("\\W+")))
        .filter(s -> !s.isBlank())
        .collect(Collectors.groupingBy(Function.identity(), Collectors.counting()));
}
Ex 14 — Parallelize calls 
ExecutorService ex = Executors.newVirtualThreadPerTaskExecutor();
List<CompletableFuture<String>> futs = ids.stream()
    .map(id -> CompletableFuture.supplyAsync(() -> svc.fetch(id), ex))
    .toList();
List<String> result = futs.stream().map(CompletableFuture::join).toList();
Ex 15 — LRU cache See Ex 9.3. Implement with `LinkedHashMap(accessOrder=true)` + `removeEldestEntry`.
Ex 16 — Race condition fix Non-safe `int n; void inc() { n++; }`. Add `synchronized`, then `AtomicInteger`. Measure.
Ex 17 — Producer-consumer `BlockingQueue` with 1 producer, 2 consumers. 1000 numbers, consumers sum them.
Ex 18 — Optional chaining Model `Order.customer.address.city` with nested `Optional`. Use `flatMap`.
Ex 19 — File walk Count `.java` files recursively, excluding `target/`.
Ex 20 — Virtual threads stress 100,000 threads sleeping 1s. Measure total time.

JVM, JDBC, JPA, SQL (21-30)

Ex 21 — `javap` reverse Disassemble a simple class, spot `iadd`, `iload`, `istore` in main.
Ex 22 — Force OOM and analyze Program filling `List`. Run with `-Xmx128m -XX:+HeapDumpOnOutOfMemoryError`. Open in Eclipse MAT.
Ex 23 — JDBC PreparedStatement H2 in-memory + `person` table. Insert 1000 records with batch + select with paging.
Ex 24 — Atomic transfer 
try (Connection c = ds.getConnection()) {
    c.setAutoCommit(false);
    try {
        ps1.executeUpdate();
        ps2.executeUpdate();
        c.commit();
    } catch (Exception e) { c.rollback(); throw e; }
}
Ex 25 — JPA entity + relation `Customer` 1-N `Order`. All LAZY. Load a Customer and its Orders with `JOIN FETCH`.
Ex 26 — N+1 in action Force N+1 loading `findAll()` of Order and iterating `o.getCustomer().getName()`. Fix with `@EntityGraph`.
Ex 27 — Spring Data derived `CustomerRepository.findByCityAndAgeBetween(String, int, int)`.
Ex 28 — SQL window function 
SELECT name, salary,
    RANK() OVER (PARTITION BY dept ORDER BY salary DESC) AS rank_in_dept
FROM employee;
Ex 29 — Index for query 1M-row table. Slow `WHERE email = ?`. `EXPLAIN ANALYZE`. Add index. Compare times.
Ex 30 — Optimistic locking Add `@Version Long version`. Two parallel updates: one fails with `OptimisticLockException`.

Ex 50 — Schedule with `@Scheduled` `@EnableScheduling` + `@Scheduled(cron = "0 0 2 * * *")` runs the job nightly. 
@Component
public class Scheduler {
    @Scheduled(cron = "0 0 2 * * *")
    public void nightly() throws Exception {
        launcher.run(importJob, params());
    }
}

How to use these exercises

  • Don't read the solutions first: try.
  • Run everything: compile, run, verify.
  • If you do 35/50 unaided, you're "professional".
  • If you do 45+ with optimizations and variations, you're "senior".
