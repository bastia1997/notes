Become a
Data Scientist

A complete path, in English, from college-level math to deploying models in production. Theory made chewable, hundreds of guided exercises, runnable code, charts and visualizations in every chapter. This is not a weekend bootcamp: it's the long road, the one that actually works.

39 sections Math + code Exercises with solutions Real notebooks ML · DL · NLP · CV

How to use

Sections follow a logical order: each chapter builds on the previous. Don't skip the math even if you already know the code — it's the difference between using scikit-learn as a black box and understanding what's happening inside. Every page contains:

  • Theory — the why, with derivations you can follow step by step.
  • Visualizations — inline charts, diagrams, rendered formulas (KaTeX).
  • Python examples — runnable code with comments and expected results.
  • Exercises — closed-ended questions, open problems, practical challenges with hidden solutions.

What you need to start

  • A PC with at least 8 GB of RAM (16 GB for serious deep learning). Mac M1+ also works great.
  • Python 3.11+, Jupyter (or JupyterLab), VS Code.
  • Conda or uv to manage environments: conda create -n ds python=3.11 numpy pandas scikit-learn matplotlib seaborn jupyter.
  • A Kaggle account (datasets and competitions) and Google Colab (free GPU).
  • Curiosity about numbers. Patience with yourself when things don't work immediately.

Philosophy

Data science is a cumulative discipline. Without linear algebra you don't understand PCA; without probability you don't understand Naive Bayes; without gradients you don't understand backpropagation. You can't fake understanding — wrong numbers always betray you. But you don't need to be a math PhD either: the right ideas applied with discipline are enough.

Sections alternate solid theory and intensive practice. Every abstract concept has at least one numerical example done by hand and one in code. Every algorithm is first implemented from scratch (to understand) and then used via library (to be efficient).

Recommended order

Follow the sections in order. If you already know a topic, read the intro anyway and do the exercises: they reveal the spots where "knowing" isn't "understanding". Estimated time for the whole path: 4–8 months at 1–2 hours per day. It's not a race.
