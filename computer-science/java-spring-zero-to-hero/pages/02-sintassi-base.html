I tipi primitivi: 8 e basta

Java ha 8 tipi primitivi. Vivono fuori dall'heap (sullo stack), non sono oggetti, non hanno metodi. Sono pure "celle di memoria".

Tipo Dimensione Range Default Uso tipico
boolean 1 bit logico (1 byte JVM) true / false false Flag
byte 8 bit -128 .. 127 0 Byte di file/protocolli
short 16 bit -32768 .. 32767 0 Raro, legacy
char 16 bit unsigned 0 .. 65535 (UTF-16 code unit) Singolo carattere
int 32 bit ~ ±2.1 miliardi 0 Default per interi
long 64 bit ~ ±9.2·10¹⁸ 0L Id grandi, timestamp
float 32 bit IEEE 754 ±3.4·10³⁸ 0.0f Raro, grafica
double 64 bit IEEE 754 ±1.8·10³⁰⁸ 0.0 Default per decimali

Regola pratica: usa int per quasi tutti gli interi e double per quasi tutti i decimali. long se serve davvero (es. timestamp in millis). Mai float se non sai esattamente perché. Mai float/double per soldi (vedi più sotto).

Suffissi e letterali

int  a = 100;
long b = 100L;          // L obbligatoria se eccede int
long c = 9_999_999_999L; // gli underscore sono leggibili, ignorati
double pi = 3.14;
float  pif = 3.14f;     // f obbligatoria
int hex = 0xFF;          // 255
int bin = 0b1010_0001;   // 161
char cc = 'A';           // apici singoli
char unicode = '€'; // € (euro)

Riferimenti (tutto il resto)

Tutto ciò che non è primitivo è un riferimento a un oggetto sull'heap.

String  nome = "Mario";          // riferimento (puntatore) a un oggetto String
Integer x    = 42;               // boxed Integer, NON primitive int
int[]   arr  = {1, 2, 3};        // riferimento a un array
LocalDate d  = LocalDate.now();  // riferimento a LocalDate

I riferimenti possono essere null. I primitivi no.

String s = null;     // ok
int    n = null;     // NON compila

Variabili: dichiarazione e scope

public class ScopeDemo {
    static int globalCounter = 0;   // campo di classe (static field)
    int instanceCounter = 0;        // campo di istanza

    public void demo() {
        int localCounter = 0;       // variabile locale
        for (int i = 0; i < 5; i++) {
            int inLoop = i * 2;     // scope: solo dentro il for
            localCounter += inLoop;
        }
        // qui `inLoop` non esiste più
        System.out.println(localCounter);
    }
}

Regola: una variabile esiste dal punto di dichiarazione alla fine del blocco { } che la contiene.

var: inferenza di tipo (Java 10+)

var name = "Mario";        // dedotto: String
var n    = 42;             // dedotto: int
var list = new ArrayList<String>();  // dedotto: ArrayList<String>

Vincoli: - Solo per variabili locali con inizializzazione. - Non per parametri di metodo, campi di classe, return type. - var n = null; NON compila (non c'è informazione di tipo). - var n; NON compila.

Usa var quando il tipo è ovvio dal valore a destra. Non per nascondere tipi complessi: chi legge deve capire al volo.

final: la variabile non si riassegna

final int MAX = 10;
MAX = 11;   // ERRORE di compilazione

final non significa "immutabile profondamente":

final List<Integer> nums = new ArrayList<>();
nums.add(1);  // ok! il riferimento è final, ma l'oggetto è mutabile
nums = new ArrayList<>();  // ERRORE: stai riassegnando

Per immutabilità vera servono classi immutabili (List.of(), Collections.unmodifiableList, record, vedremo).

Operatori

Aritmetici

int a = 17, b = 5;
System.out.println(a + b);   // 22
System.out.println(a - b);   // 12
System.out.println(a * b);   // 85
System.out.println(a / b);   // 3   <-- divisione INTERA (tronca)
System.out.println(a % b);   // 2   <-- modulo
System.out.println((double) a / b);  // 3.4  <-- forzando double

Trappola: la divisione fra int tronca. (int)(7/2) == 3. Per il decimale devi avere almeno un operando in floating point.

Compound assignment

int x = 10;
x += 5;   // x = 15
x -= 2;   // x = 13
x *= 3;   // x = 39
x /= 4;   // x = 9
x %= 5;   // x = 4

Incremento

int i = 5;
int a = i++;  // a = 5, poi i diventa 6 (post-incremento)
int j = 5;
int b = ++j;  // j diventa 6, poi b = 6 (pre-incremento)

Bitwise

int x = 0b1100;
int y = 0b1010;
System.out.println(Integer.toBinaryString(x & y));  // 1000 (AND)
System.out.println(Integer.toBinaryString(x | y));  // 1110 (OR)
System.out.println(Integer.toBinaryString(x ^ y));  // 0110 (XOR)
System.out.println(Integer.toBinaryString(~x));     // ...11110011 (NOT)
System.out.println(Integer.toBinaryString(x << 2)); // 110000 (shift sinistro)
System.out.println(Integer.toBinaryString(x >> 1)); // 110 (shift destro, conserva segno)
System.out.println(Integer.toBinaryString(x >>> 1));// 110 (shift destro, riempie con 0)

Confronto

a == b   // uguaglianza (per primitivi: valori; per oggetti: STESSO riferimento)
a != b
a < b, a <= b, a > b, a >= b

== tra oggetti confronta i riferimenti, non i contenuti. Per confrontare contenuti usi .equals().

String s1 = new String("ciao");
String s2 = new String("ciao");
System.out.println(s1 == s2);       // false (riferimenti diversi)
System.out.println(s1.equals(s2));  // true  (contenuti uguali)

Logici

a && b   // AND (short-circuit: se a è false, b non viene valutato)
a || b   // OR  (short-circuit: se a è true, b non viene valutato)
!a       // NOT

Lo short-circuit ti salva da NullPointerException:

if (s != null && s.length() > 0) { ... }  // ok
if (s.length() > 0 && s != null) { ... }  // BOOM se s è null

Ternario

int x = (a > b) ? a : b;
String msg = (n == 1) ? "1 item" : n + " items";

Overflow: perché un numero positivo può diventare negativo

int ha range -2³¹ .. 2³¹-1 (~ ±2.1 miliardi). Se sfondi, ricicli (wrap-around silenzioso):

int big = Integer.MAX_VALUE;     // 2147483647
System.out.println(big + 1);     // -2147483648   <-- overflow silenzioso!

Java non lancia eccezioni per overflow di interi. È la fonte di bug nei calcoli finanziari mal scritti.

Soluzioni: 1. Usa long. Funziona finché stai sotto 9·10¹⁸. 2. Usa Math.addExact, Math.multiplyExact, ... che lancia ArithmeticException se va in overflow: java int sum = Math.addExact(big, 1); // ArithmeticException: integer overflow 3. Per soldi e calcoli precisi: BigDecimal. Sempre. Vedi sezione 6.

Floating point: perché 0.1 + 0.2 non fa 0.3

System.out.println(0.1 + 0.2);   // 0.30000000000000004
System.out.println(0.1 + 0.2 == 0.3);  // false

double è IEEE 754: non rappresenta esattamente 0.1 in binario (è infinito periodico, come 1/3 in decimale). Non usare double per soldi, mai. Usa BigDecimal:

import java.math.BigDecimal;
BigDecimal a = new BigDecimal("0.1");
BigDecimal b = new BigDecimal("0.2");
System.out.println(a.add(b));   // 0.3 esatto

Nota: passa stringhe al costruttore, non double. new BigDecimal(0.1) è già "rotto" perché 0.1 è già impreciso.

Controllo di flusso

if / else

if (saldo > 1000) {
    livello = "GOLD";
} else if (saldo > 100) {
    livello = "SILVER";
} else {
    livello = "BRONZE";
}

switch classico

String giorno = switch (n) {
    case 1: yield "Lun";    // syntax con yield (Java 14+)
    case 2: yield "Mar";
    default: yield "Sconosciuto";
};

switch espressione "freccia" (Java 14+)

String giorno = switch (n) {
    case 1 -> "Lunedì";
    case 2 -> "Martedì";
    case 3, 4, 5 -> "Lavorativo";
    case 6, 7 -> "Weekend";
    default -> "Sconosciuto";
};

Lo switch "freccia" non ha fall-through: non serve break. Più sicuro.

Pattern matching nello switch (Java 21)

sealed interface Shape permits Circle, Square {}
record Circle(double r) implements Shape {}
record Square(double side) implements Shape {}

double area(Shape s) {
    return switch (s) {
        case Circle c -> Math.PI * c.r() * c.r();
        case Square sq -> sq.side() * sq.side();
    };
}

Lo vediamo in dettaglio nella Sezione 18.

while e do-while

int i = 0;
while (i < 5) {
    System.out.println(i);
    i++;
}

int j = 0;
do {
    System.out.println(j);
    j++;
} while (j < 5);   // viene eseguito almeno una volta

for classico

for (int i = 0; i < 10; i++) {
    System.out.println(i);
}

for-each (sintatticamente for-in)

int[] nums = {1, 2, 3, 4, 5};
for (int n : nums) {
    System.out.println(n);
}

List<String> words = List.of("ciao", "mondo");
for (String w : words) {
    System.out.println(w);
}

break e continue

for (int i = 0; i < 100; i++) {
    if (i == 50) break;        // esce dal loop
    if (i % 2 == 0) continue;  // salta al prossimo i
    System.out.println(i);     // stampa solo dispari < 50
}

break con label (raro, ma esiste)

outer:
for (int i = 0; i < 10; i++) {
    for (int j = 0; j < 10; j++) {
        if (i * j > 30) break outer;  // esce da ENTRAMBI i loop
    }
}

Esercizi

Es 2.1 — Overflow esplicito Scrivi un programma che: 1. Stampa `Integer.MAX_VALUE`. 2. Calcola `Integer.MAX_VALUE + 1` e stampa il risultato (vedrai negativo). 3. Usa `Math.addExact` per fare la stessa somma. Cattura l'eccezione e stampa "overflow". 
public class Overflow {
    public static void main(String[] args) {
        int max = Integer.MAX_VALUE;
        System.out.println("max = " + max);
        System.out.println("max + 1 (silenzioso) = " + (max + 1));
        try {
            int r = Math.addExact(max, 1);
            System.out.println("r = " + r);
        } catch (ArithmeticException e) {
            System.out.println("overflow detected!");
        }
    }
}
Es 2.2 — Soldi giusti Implementa una funzione `aggiungiIva(BigDecimal prezzo, BigDecimal aliquota)` che restituisce il prezzo IVA inclusa. Aliquota in formato `0.22` per il 22%. 
import java.math.BigDecimal;
import java.math.RoundingMode;

public class Iva {
    public static BigDecimal aggiungiIva(BigDecimal prezzo, BigDecimal aliquota) {
        return prezzo.multiply(BigDecimal.ONE.add(aliquota))
                     .setScale(2, RoundingMode.HALF_UP);
    }

    public static void main(String[] args) {
        BigDecimal p = new BigDecimal("19.99");
        BigDecimal iva = new BigDecimal("0.22");
        System.out.println(aggiungiIva(p, iva));  // 24.39
    }
}
Punti chiave: passa stringhe, usa `.multiply` non `*`, usa `setScale` per arrotondare a 2 decimali.
Es 2.3 — FizzBuzz (il classico) Stampa i numeri da 1 a 30. Per i multipli di 3 stampa "Fizz", per i multipli di 5 "Buzz", per i multipli di entrambi "FizzBuzz". 
public class FizzBuzz {
    public static void main(String[] args) {
        for (int i = 1; i <= 30; i++) {
            String out;
            if (i % 15 == 0) out = "FizzBuzz";
            else if (i % 3 == 0) out = "Fizz";
            else if (i % 5 == 0) out = "Buzz";
            else out = String.valueOf(i);
            System.out.println(out);
        }
    }
}
Versione "moderna" con switch (Java 21): 
String out = switch (0) {
    case int z when i % 15 == 0 -> "FizzBuzz";  // non funziona così — vedi esercizio avanzato
    // ...
};
Lo switch su int+guard arriverà nella sezione 18.
Es 2.4 — `==` vs `.equals()` Indovina l'output: 
String a = "ciao";
String b = "ciao";
String c = new String("ciao");

System.out.println(a == b);
System.out.println(a == c);
System.out.println(a.equals(c));
Risposta: - `a == b` → **true**. Le stringhe letterali sono "interned": stesso oggetto in memoria. - `a == c` → **false**. `new String("...")` crea un nuovo oggetto sull'heap. - `a.equals(c)` → **true**. Confronto di contenuto. Lezione: **per stringhe usa sempre `.equals()`**, mai `==`.
Es 2.5 — Loop con due indici Stampa una matrice 5×5 dove ogni cella contiene `i*j`. Allinea le colonne (usa `%4d` con `printf`). 
public class Matrix {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            for (int j = 1; j <= 5; j++) {
                System.out.printf("%4d", i * j);
            }
            System.out.println();
        }
    }
}

Cosa devi portarti via

  • 8 primitivi, tutto il resto è riferimento. null solo per riferimenti.
  • int e double come default. Mai double per soldi: usa BigDecimal.
  • == confronta riferimenti per oggetti. Usa .equals().
  • Overflow di int è silenzioso. Usa Math.addExact o long/BigInteger.
  • switch espressione "freccia" è quasi sempre meglio del classico.
  • var solo per locali, quando il tipo è ovvio dal valore.
